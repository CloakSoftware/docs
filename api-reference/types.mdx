These are the data types you receive in event payloads and from `Client` methods. The SDK ships full TypeScript declarations, so your editor knows all of this already.

## Message

The payload of a [`messageCreate`](/api-reference/events#messagecreate) event. It carries the decrypted content, who sent it, where it came from, and helpers to reply.

```ts
interface Message {
  content: string;
  authorId: string;
  authorName: string;
  messageId: string;
  serverId: string;
  channelId: string;
  reply(text: string): Promise<void>;
  channel: { send(text: string): Promise<void> };
}
```

<ResponseField name="content" type="string">
  The decrypted message text. Empty string if the key was missing or decryption failed, so check for content before acting on it.
</ResponseField>

<ResponseField name="authorId" type="string">
  The sender's normalized id. Compare with `client.user?.id` to detect your bot's own messages.
</ResponseField>

<ResponseField name="authorName" type="string">
  The sender's display name.
</ResponseField>

<ResponseField name="messageId" type="string">
  The unique id of this message.
</ResponseField>

<ResponseField name="serverId" type="string">
  The server the message came from.
</ResponseField>

<ResponseField name="channelId" type="string">
  The channel the message came from.
</ResponseField>

<ResponseField name="reply" type="(text: string) => Promise<void>">
  Sends a reply into the same channel this message arrived in. The location is already known, so you pass only the text.
</ResponseField>

<ResponseField name="channel.send" type="(text: string) => Promise<void>">
  Sends into the same channel. Equivalent to `reply`, phrased as sending to the channel rather than replying to a message.
</ResponseField>

```ts
client.on('messageCreate', (msg) => {
  if (msg.content.trim() === '!ping') {
    msg.reply('pong');           // reply to the message
    // or: msg.channel.send('pong'); // same destination
  }
});
```

## SystemMessageEvent

The payload of a [`systemMessage`](/api-reference/events#systemmessage) event.

```ts
interface SystemMessageEvent {
  type: 'member_join' | 'bot_add' | 'unknown';
  actorId: string;
  actorName: string;
  serverId: string;
  channelId: string;
  groupId?: string;
}
```

<ResponseField name="type" type="'member_join' | 'bot_add' | 'unknown'">
  Which system event this is. `member_join` when someone joins, `bot_add` when a bot is added, `unknown` for anything the SDK does not recognize.
</ResponseField>

<ResponseField name="actorId" type="string">
  The id of the member the event is about, such as the person who joined.
</ResponseField>

<ResponseField name="actorName" type="string">
  The display name of that member.
</ResponseField>

<ResponseField name="serverId" type="string">
  The server the event happened in.
</ResponseField>

<ResponseField name="channelId" type="string">
  The channel the event is associated with.
</ResponseField>

<ResponseField name="groupId" type="string">
  The group of the source channel, where known. Pass it to `send()` so the message lands in the right channel, even one the bot has not seen traffic from yet. May be undefined if the channel's group has not been seen.
</ResponseField>

## Guild

The payload of [`guildCreate`](/api-reference/events#guildcreate) and [`guildDelete`](/api-reference/events#guilddelete). A server is called a guild here, matching the discord.js naming.

```ts
interface Guild {
  id: string;
}
```

## BotUser

Your bot's identity, available as `client.user` after login.

```ts
interface BotUser {
  id: string;
  username: string;
}
```

<ResponseField name="id" type="string">
  Your bot's normalized id. Compare it against `msg.authorId` to ignore your bot's own messages.
</ResponseField>

<ResponseField name="username" type="string">
  Your bot's username.
</ResponseField>

## PermissionSet

The resolved capability set for one server, returned by [`permissions()`](/api-reference/client#permissions) and included in the [`permissionsUpdate`](/api-reference/events#permissionsupdate) event.

```ts
type PermissionSet = { effectiveRank: number } & Record<PermissionName, boolean>;
```

It is one boolean per permission, plus a numeric rank.

<ResponseField name="effectiveRank" type="number">
  The bot's effective rank in the server.
</ResponseField>

<ResponseField name="[permission]" type="boolean">
  One boolean per [permission name](/api-reference/permissions), such as `message_send` or `member_ban`. `true` means the bot holds it in this server.
</ResponseField>

```ts
const set = client.permissions(serverId);
if (set?.message_send) {
  // the bot may send here
}
```
